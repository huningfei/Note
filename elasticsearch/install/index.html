    es6.6安装单台环境¶

    安装java环境¶

    • jdk安装，最好是在1.8以上，下面的路径根据实际情况而定
    解压：tar zxvf jdk-8u60-linux-i586.tar.gz
  移动 ：  sudo  mv jdk1.8.0_60  /usr/local/
sudo   vim /etc/profile.d/java.sh加入这几行
JAVA_HOME=/usr/local/jdk1.8.0_60
JAVA_BIN=/usr/local/jdk1.8.0_60/bin
JRE_HOME=/usr/local/jdk1.8.0_60/jre
PATH=$PATH:/usr/local/jdk1.8.0_60/bin:/usr/local/jdk1.8.0_60/jre/bin
CLASSPATH=/usr/local/jdk1.8.0_60/jre/lib:/usr/local/jdk1.8.0_60/lib:/usr/local/jdk1.8.0_60/jre/lib/charsets.jar

初始化  source  /etc/profile.d/java.sh

最后运行：java -version 会出来版本信息

    安装es¶

    • 下载，解压，并更改配置文件
    grep -v "^#" elasticsearch.yml |grep -v "^$"
cluster.name: test # 集群名称
node.name: node-1 # 节点名
path.data: /home/yx/server/elasticsearch/data #数据目录
path.logs: /home/yx/server/elasticsearch/logs # 日志目录
bootstrap.memory_lock: false
bootstrap.system_call_filter: false # 为了解决启动失败添加的
network.host: 0.0.0.0  #本机ip
http.port: 9200 #端口号
http.cors.enabled: true   # 是为了head插件可以访问es
http.cors.allow-origin: "*" # 是为了head插件可以访问es

    启动方法¶

    • 启动
    /home/yx/server/elasticsearch/bin/elasticsearch #前台启动
/home/yx/server/elasticsearch/bin/elasticsearch -d #后台启动

    启动报错处理¶

    • max file descriptors [4096] for elasticsearch process is too low, increase to at least [65536]

    解决方案： 切到root 用户：进入到security目录下的limits.conf；执行命令
vim /etc/security/limits.conf 在文件的末尾添加下面的参数值：


* soft nofile 65536
* hard nofile 131072
* soft nproc 4096
* hard nproc 4096
前面的*符号必须带上，用户重新登录就可以了。执行完成后可以使用命令 ulimit -n 查看进程数 。
ulimit -Hu
ulimit -Su
    - [2]: max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]

    解决方案：切换到root用户修改配置sysctl.conf
sudo vim /etc/sysctl.conf
增加配置值： vm.max_map_count=655360
sudo sysctl -p
    • ERROR: bootstrap checks failed system call filters failed to install; check the logs and fix your configuration or disable system call filters at your own risk
    问题原因：因为Centos6不支持SecComp，而ES5.2.1默认bootstrap.system_call_filter为true进行检测，所以导致检测失败，失败后直接导致ES不能启动。详见 ：https://github.com/elastic/elasticsearch/issues/22899



解决方法：在elasticsearch.yml中配置bootstrap.system_call_filter为false，注意要在Memory下面:
bootstrap.memory_lock: false
bootstrap.system_call_filter: false  # 默认这行是没有的，需要手动添加

    安装head插件¶

    • 安装Node.js
    https://npm.taobao.org/mirrors/node   下载地址


wget https://npm.taobao.org/mirrors/node/v10.15.0/node-v10.15.0-linux-x64.tar.xz

tar -xf node-v10.15.0-linux-x64.tar.xz

sudo mv node-v10.15.0-linux-x64/ /usr/local/node

 配置并生效

sudo vim /etc/profile
export NODE_HOME=/usr/local/node
export PATH=$PATH:$NODE_HOME/bin

source /etc/profile

查看版本验证
yx@php:~/src/es$ node -v
v10.15.0
yx@php:~/src/es$ npm -v
6.4.1
    • 下载head插件
    sudo apt-get install git -y

git clone https://github.com/mobz/elasticsearch-head.git
    • 设置淘宝仓库安装cnpm
    npm install -g cnpm --registry=https://registry.npm.taobao.org
    • 安装grunt
    # 进去刚才下载的head插件目录里面
cd elasticsearch-head/


cnpm install    # 安装各种依赖插件
    • 启动head插件服务
     第一种启动方式:
 找对你的 elasticsearch-head/目录所在的路径

进入到 /home/yx/server/elasticsearch-head-master 目录下然后启动
#后台启动
node_modules/grunt/bin/grunt server &
  # 查看是否启动
yx@php:~/src/es/elasticsearch-head/node_modules$ sudo netstat -lnp | grep 9100
tcp        0      0 0.0.0.0:9100            0.0.0.0:*               LISTEN      8760/grunt  

关闭 ps aux | grep grun  查出id,然后kill id

第二种启动方式：
npm run start &
